Run Concordion Test with NUnit

Similar to the Java version of Concordion it integrates with one of the most popular testing frameworks. Concordion-NET provides an NUnit-Addin that allows executing Concordion tests with any NUnit environment that supports NUnit-Addins.

Please note that at the time of writing the NUnit Test Adapter for Visual Studio needs an update to fully support addins. Nevertheless, the Concordion-NUnit-Addin works fine with the standalone NUnit runner. Thus, we are describing here how to run Concordion tests with the standard NUnit GUI runner.

  1. First download and install NUnit.
  2. If you have not already done, download Concordion.
  3. Take the Concordion.NUnit.dll and copy it to the addin directory of our NUnit installation (<NUnit-installation-path>/bin/addins).
  4. Add a reference to Concordion.dll in your project. You find the Concordion.dll and the necessary OGNL.dll in the package you have downloaded in step 2.
  5. Build your Concordion project containing your Concordion tests.
  6. Finally, execute your Concordion tests like any "usual" NUnit tests with the runner of your choice.

Hopefully, you will see the typical green bar, when your tests pass. :-)

You find the Concordion test reports in the output directory that can be specified in the Concordion configuration. By default Concordion-NET places them into the temp directory (on Windows defined by the environment variable %Temp%, which by default points to C:\Users\<your-windows-user>\AppData\Local\Temp).

You can debug your Concordion tests in the same way as debugging NUnit tests.
